Git 命令很多，但日常开发真正反复用到的并不多。这是我给自己整理的一张小抄。

## 查看现场

```bash
git status
git diff
git log --oneline -8
```

动手前先看一眼现场，常常能省下十分钟的慌张。

## 保存一次清楚的修改

```bash
git add src/app.js
git commit -m "feat: add article search"
```

我倾向于一次提交只说明一件事。以后寻找问题时，历史记录会像一本清楚的日记。

## 临时收起未完成的工作

```bash
git stash push -m "half-finished search UI"
git stash pop
```

适合临时切去修一个紧急问题，但别把 stash 当成长期储藏室——那里很容易变成数字阁楼。

## 撤销还没提交的单个文件

```bash
git restore path/to/file.js
```

运行前一定再确认一次文件名，因为未提交的修改会消失。

## 最重要的习惯

命令忘了可以查，**提交前看 diff** 才是最值得养成的习惯。它既能发现误改，也能帮我把提交信息写得更准确。
