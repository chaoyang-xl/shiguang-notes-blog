搭一个自己的博客，真正困难的部分往往不是代码，而是**决定开始写**。这篇笔记记录本站的技术选择，也留作之后继续改造的地图。

## 为什么选择静态博客

我的需求很简单：写学习笔记、贴代码、偶尔记录生活。静态网站刚好有几个很讨喜的特点：

- 文件就是内容，迁移和备份都很轻松；
- 不需要数据库，几乎没有维护成本；
- GitHub Pages 可以免费托管；
- 页面快，搜索引擎也容易读取。

> 工具应该让写作变简单，而不是让维护工具本身成为新的工作。

## 一篇文章是怎样出现的

每篇文章都是 `posts` 目录下的一个 Markdown 文件。文件开头的几行叫作 front matter，用来告诉博客标题、日期和标签。

```markdown
---
title: 今天学会了什么
date: 2026-06-27
description: 一句话介绍这篇笔记。
tags: [学习, 随笔]
---

正文从这里开始。
```

运行构建命令之后，脚本会读取所有文章，把 Markdown 转换成浏览器能显示的 HTML：

```bash
npm run build
```

### 小而清楚的目录结构

```text
posts/          # 所有 Markdown 文章
assets/         # 样式和交互
scripts/        # 构建与本地预览
dist/           # 最终生成的网站
```
